DuckDB Vector Search Extension (IVF-PQ)

A high-performance C++ extension that brings native Approximate Nearest Neighbor (ANN) search to DuckDB. It implements Inverted File Indexing with Product Quantization (IVF-PQ) to enable memory-efficient similarity search on millions of embeddings directly within SQL queries.

Full Description

This project is a custom extension for DuckDB designed to handle the growing need for vector analytics in local databases. Unlike external vector databases, this extension runs inside the DuckDB process, allowing for zero-copy data access and seamless integration with existing SQL tables.

I engineered the core indexing logic using IVF-PQ (Inverted File + Product Quantization), which clusters vectors into Voronoi cells and compresses them using subspace quantization. This reduces memory usage by up to 32x compared to raw vectors while maintaining high recall.

A standout feature is the custom Query Optimizer. I implemented a rewrite rule that intercepts SQL queries with WHERE clauses (e.g., WHERE category = 'electronics') and pushes them down into the vector search scanner. This enables Hybrid Searchâ€”filtering candidates before performing expensive distance calculationsâ€”drastically improving performance for filtered queries.

Key Features

  • IVF-PQ Indexing: Implements K-Means clustering and Product Quantization for efficient storage and retrieval.
  • Hybrid Search: Custom optimizer rules support predicate pushdown for metadata filtering.
  • Native Storage: Persists index data and centroids as standard, inspectable DuckDB tables.
  • Distance Metrics: Optimized L2 (Euclidean) distance calculations.

SmartDoc Finder: A Self-Hosted RAG Search Engine

A complete, privacy-first Retrieval-Augmented Generation (RAG) system built from the ground up to answer complex questions using a private collection of documents.

The project is architected as a full-stack, event-driven microservices application, containerized with Docker Compose for seamless one-command startup. It features a sophisticated 3-stage pipeline that combines hybrid search (Apache Lucene + FAISS), a cross-encoder re-ranker for relevance, and a locally-hosted LLM (via Ollama) to generate accurate, verifiable answers while ensuring 100% data privacy.

Weenix: A UNIX-Like Operating System

Engineered the core kernel of a UNIX-like OS from scratch, implementing robust modules for process and thread management, scheduling, and thread-safe synchronization using mutexes.

Developed a flexible virtual file system (VFS) with support for multiple underlying file systems, providing a unified and reliable interface for all file operations. Designed and built a virtual memory manager to handle user address spaces and system calls, enabling efficient and protected user process execution.

Image Classification Using Transfer Learning

This project builds an image classifier using transfer learning with pre-trained models like ResNet50, EfficientNetB0, and VGG16. It fine-tunes only the final layer while freezing earlier layers to extract meaningful features.

Data augmentation (cropping, rotation, flipping) enhances generalization, and the model is trained with ReLU activation, batch normalization, dropout, and ADAM optimizer for at least 50 epochs with early stopping. The goal is to create a robust classifier optimized for small datasets using state-of-the-art deep learning techniques.
